# Specs

Each spec is one markdown file describing one change. Use
[TEMPLATE.md](TEMPLATE.md) as the starting point.

## Conventions
- **Filename:** `NNNN-short-name.md` (e.g. `0001-cloud-multi-user-install.md`)
- **One spec per change** — split big efforts into multiple specs
- **Keep specs short** — link out to deeper context rather than inlining it
- **Status flow:** Draft → Approved → In Progress → Shipped (or Abandoned)
