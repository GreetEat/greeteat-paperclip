## Spec-driven development in one paragraph
Before writing code, we write a spec describing the problem, the proposed
solution, and how we'll know it works. Specs live in `specs/` and follow
[TEMPLATE.md](specs/TEMPLATE.md). Code reviews check that the implementation
matches the spec.
