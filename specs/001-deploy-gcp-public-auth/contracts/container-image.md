# Contract: Paperclip Container Image

**Built by**: `.github/workflows/build-image.yml`
**Stored at**: Artifact Registry, `<region>-docker.pkg.dev/<project>/paperclip/paperclip:<digest>`
**Consumed by**: Cloud Run service (`infra/modules/compute/`) and Cloud Run Job (`infra/modules/jobs/`)
**Source**: `infra/docker/Dockerfile` building from a pinned upstream Paperclip git tag

This contract defines what the Paperclip container image accepts as
input and exposes as output. Both the Cloud Run service and the doctor
Cloud Run Job use this same image.

## Build inputs (build-time, baked into image)

| Input | Source | Notes |
|---|---|---|
| Paperclip release tag | `infra/envs/prod/versions.tfvars` (`paperclip_version`) | Pinned for the single environment. CI clones Paperclip at this exact tag and builds against it. |
| Node.js base image | Pinned in `Dockerfile` | Must match Paperclip's documented Node version requirement; updated in lockstep with Paperclip upgrades. |
| Build-time secrets | None | The image MUST contain zero secret material. Build context never receives master keys, DB passwords, LLM keys, or HMAC credentials. |

## Runtime inputs (env vars, injected at deploy time)

### Required from Secret Manager (mounted via Cloud Run `secret_key_ref`)

| Env var | Purpose |
|---|---|
| `PAPERCLIP_SECRETS_MASTER_KEY` | 32-byte base64 master key for Paperclip's encrypted secret store. Generated by `bootstrap-master-key.sh`. |
| `DATABASE_URL` | `postgres://…` connection string for Cloud SQL via Serverless VPC Connector. |
| `S3_ACCESS_KEY_ID` | GCS HMAC interop access ID (for the S3-compatible storage backend). |
| `S3_SECRET_ACCESS_KEY` | GCS HMAC interop secret. |

**No `ANTHROPIC_API_KEY` is set in any form.** Verified on 2026-04-10:
Paperclip's `claude_local` adapter accepts the unset state (logs
"ANTHROPIC_API_KEY is not set; subscription-based auth can be used")
and the spawned Claude Code authenticates to Vertex AI directly via
the Cloud Run service account when `CLAUDE_CODE_USE_VERTEX=1` is set.
No stub, no real key.

### Required from plain Cloud Run env (set in service spec)

| Env var | Value | Purpose |
|---|---|---|
| `PORT` | Injected by Cloud Run | Cloud Run sets this; Paperclip's `PORT` env var honors it directly. |
| `HOST` | `0.0.0.0` | Override Paperclip's default `127.0.0.1` so Cloud Run's load balancer can reach the container. |
| `PAPERCLIP_DEPLOYMENT_MODE` | `public` | Forces `authenticated` + `public` mode; required by FR-001. |
| `PAPERCLIP_PUBLIC_URL` | `https://<your-domain>` | Explicit public URL required by Paperclip's public mode. The exact hostname is set in tfvars by the operator. |
| `PAPERCLIP_SECRETS_STRICT_MODE` | `true` | Forces secret-reference mandate per principle IV. |
| `PAPERCLIP_INSTANCE_ID` | `prod` | Identifies the deployment instance in logs and metrics. Single environment for v1. |
| `S3_ENDPOINT` | `https://storage.googleapis.com` | GCS S3 interop endpoint. |
| `S3_BUCKET` | `paperclip-492823-uploads` | The deployment's GCS bucket name. |
| `S3_REGION` | `auto` (or the bucket's region) | Required by some S3 SDK clients; GCS interop accepts `auto`. |
| `LOG_LEVEL` | `info` | Controls Paperclip log verbosity. |
| `LOG_FORMAT` | `json` | Required for Cloud Logging structured-log mapping. |
| `CLAUDE_CODE_USE_VERTEX` | `1` | Switches the Claude Code CLI (spawned by Paperclip's `claude_local` adapter) to Vertex AI instead of api.anthropic.com. |
| `CLOUD_ML_REGION` | `global` | Vertex AI global endpoint — no pricing premium, dynamic routing across regions where Claude has capacity. |
| `ANTHROPIC_VERTEX_PROJECT_ID` | `paperclip-492823` | GCP project ID for Vertex AI Anthropic model invocations. |
| `ANTHROPIC_DEFAULT_SONNET_MODEL` | `claude-sonnet-4-6` | Pin Claude Code's Sonnet model to the version we verified live in Model Garden. |

## Runtime outputs

| Output | Where | Notes |
|---|---|---|
| HTTP listener | `0.0.0.0:$PORT` | Single port serves the React UI, REST API, and `/health`. |
| stdout / stderr | Captured by Cloud Run → Cloud Logging | Paperclip MUST emit JSON; one event per line; no secret material in any log line at any verbosity. |

## Boot contract (entrypoint.sh)

The image's entrypoint MUST, in order:

1. Verify every required env var is present. **Refuse to start** with a
   non-zero exit if any required var is missing or empty.
2. Verify `DATABASE_URL` is reachable; refuse to start if not.
3. Run Paperclip's Drizzle migrations. Refuse to start if migrations fail
   (do not roll forward into the server with a stale schema).
4. `exec` Paperclip's server process so it becomes PID 1 and receives
   signals correctly.

## Cloud Run Job mode (doctor)

The same image, when launched as a Cloud Run Job, runs `paperclipai doctor`
instead of the server. The job overrides the entrypoint command:

- `command: ["paperclipai"]`
- `args: ["doctor"]`

Doctor MUST exit non-zero on any failure so the calling
`gcloud run jobs execute --wait` reports failure to the deploy script.

## Image hygiene

- Distroless or minimal base where possible (Paperclip's Node version
  permitting).
- Non-root user inside the container.
- No package managers in the final stage.
- No secrets, no `.env` files, no service account JSON keys, ever.
- Trivy scan in CI must pass HIGH and CRITICAL with no unfixed findings.
- Hadolint must pass with the project's pinned ruleset.

## Versioning

The image is identified to the deployment **only** by its immutable
digest (`sha256:…`). Tags exist for human inspection
(`<paperclip-version>`, `<git-sha>`) but are never referenced by the
Terraform service spec.
